Microsoft Solutions Framework

É um conjuto de prínipios, modelos, disciplinas, conceitos e diretrizes para fornecer serviçõs de tecnologia
da informação da Microsoft.
MSF não se limita apenas não se limita apenas ao desenvolvimento dos aplicativos; também é aplicável a outros
projetos de implatação, redes ou infraestruturas. O MSF não obriga desemvolver a usar uma metodologia específica(como um modelo em
cascata ou o desenvolvimento ágil de software.

Fundação da MSF

MSF foiintroduzido pela primeira vez pela Microsoft como versão1.0 em 1993, e uma versão 2.0 foi lançada em 1997.
em 2002 o MSF versão 3.0 foi lançado.Ele modificou a versão 2.0.

    Modificação do MSF

  • Modelos previamente separados combinados em modelos unificados de equipes e processo projetados para
    em varios tipos de projetos, incluindo implatação, integração de software empresarial e projetos de
    desenvolvimento.

  • Dobrou os modelos de Desenvolvimento de Aplicativos e Implantação de Infraestrutura em um único Modelo de
    Processo composto por cinco fases.

  • Adicionadas Disciplinas de Gerenciamento de Projetos e Gerenciamento de Preparação.

  • Fez alterações na Disciplina de Gerenciamento de Riscos.

  • Adicionados links entre o MSF e o Microsoft Operations Framework (MOF).

  • Adicionado um Programa de Praticantes de MSF projetado para treinar indivíduos para liderar ou participar de projetos de MSF.

    • MSF Versão 4.0

    O MSF versão 4.0 foi lançado em 2005. O lançamento foi uma grande atualização do Modelo de Processo (agora chamado de Modelo de Governança)
    e do Modelo de Equipe. O MSF 4.0 incluiu técnicas para duas metodologias separadas: MSF para Desenvolvimento Ágil de Software (MSF Agile) e MSF para Melhoria de Processos CMMI (MSF4CMMI).

    MSF vantagens

    1. comunicação aberta e transparente entre todos os envolvidos;
    2. visão compartilhada do negócio;
    3. equipe capacitada;
    4. atribuição de papéis e responsabilidades desde o início do projeto;
    5. entregas incrementais;
    6. flexibilidade para mudar sempre que necessário;
    7. qualidade das entregas;
